# tbqh — Personalized Group Discussion Prompts

A full-stack MVP for generating personalized discussion questions for groups, powered by AI.

## Tech Stack

- **Frontend**: Next.js 14 (App Router) + Tailwind CSS
- **Backend**: FastAPI (Python 3.12) + SQLAlchemy + Alembic
- **Database**: PostgreSQL 16 + pgvector
- **AI**: OpenAI-compatible LLM + Embeddings

## Quick Start

### Prerequisites

- Docker & Docker Compose
- An OpenAI API key (or compatible provider)

### 1. Set environment variables

```bash
export LLM_API_KEY=sk-your-key-here
# Optional:
# export LLM_MODEL=gpt-4.1-mini
# export EMBEDDINGS_MODEL=text-embedding-3-small
```

### 2. Start all services

```bash
cd infra
docker compose up --build
```

This launches:
- **db** (Postgres + pgvector) on `:5432`
- **api** (FastAPI) on `:8000`
- **web** (Next.js) on `:3000`

Alembic migrations run automatically on API startup.

### 3. Seed the question dataset

```bash
# From within the api container:
docker exec -it app_api python /app/../scripts/import_questions.py

# Then embed them (requires LLM_API_KEY):
docker exec -it app_api python /app/../scripts/embed_questions.py
```

Or from the host (if you have Python + deps installed):
```bash
DATABASE_URL=postgresql+psycopg://app:app@localhost:5432/app \
  python scripts/import_questions.py

DATABASE_URL=postgresql+psycopg://app:app@localhost:5432/app \
  LLM_API_KEY=sk-your-key \
  python scripts/embed_questions.py
```

### 4. Open the app

- Frontend: http://localhost:3000
- API docs: http://localhost:8000/docs
- Health check: http://localhost:8000/health

## User Flow

1. **Sign in** with email magic link (in dev, the link is printed to the API console)
2. **Create a session** from the dashboard
3. **Describe your group** in free text
4. **Answer follow-up questions** generated by the AI
5. **View discussion questions** one by one — Like ❤️, Dislike 👎, or Skip ⏭️
6. **Get 10 more** at any time
7. **Check your favorites** and blocked list

## Project Structure

```
apps/
  api/          # FastAPI backend
    app/
      api/v1/routes/   # Route handlers
      core/            # Config, security, logging
      db/              # Models, migrations
      services/        # LLM, embeddings, recommend, intake
    requirements.txt
    alembic.ini
  web/          # Next.js frontend
    app/               # Pages (App Router)
    lib/               # API client
infra/
  compose.yml          # Docker Compose
  docker/              # Dockerfiles
scripts/
  import_questions.py  # Seed dataset
  embed_questions.py   # Compute embeddings
openapi/
  openapi.yaml         # API specification
schemas/
  *.json               # LLM output JSON schemas
```
